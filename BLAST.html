BUILD SMARTER

O Framework BLAST é um sistema de 5 etapas que elimina erros lógicos no desenvolvimento com IA — do blueprint ao deploy, tudo conversando com a ferramenta.

SCROLL PARA EXPLORAR
// 01 — O Framework

5 Etapas.
Zero Erros Lógicos.

B
Blueprint
// PLANO
Define a lógica em linguagem natural antes de escrever uma linha de código. Responde às 5 perguntas essenciais que guiam todo o projeto.

O Blueprint funciona como a fundação do projeto. Sem ele, a IA pode tomar decisões que parecem certas localmente, mas quebram o sistema globalmente.

🎯 North Star — objetivo principal 🔗 Serviços externos 🗄️ Fonte da verdade 📦 Formato de entrega 📋 Regras de comportamento
L
Links
// CONECTIVIDADE
Testa todas as conexões de API e credenciais antes de começar a codificar. O "handshake" que garante que tudo vai se comunicar.

Validar conexões antes de construir evita horas de debug. Se uma API falha no final, você perdeu todo o esforço anterior.

✅ Teste de APIs 🔑 Validação de credenciais 🤝 Handshake verification ⏱️ @ min 09:30
A
Architecture
// ARQUITETURA
Divide o app em três camadas distintas: SOPs, Navegação e Ferramentas. Cada camada tem uma responsabilidade única e bem definida.

A separação em camadas é o que torna o sistema previsível. A IA nunca fica "adivinhandou" — ela segue regras claras em cada nível.

📝 Camada 1 — SOPs em Markdown 🧠 Camada 2 — Raciocínio da IA 🐍 Camada 3 — Scripts Python ⏱️ @ min 21:31
S
Stylize
// ESTILIZAÇÃO
Refina o design usando referências visuais e componentes prontos. A IA imita o estilo de interfaces existentes para criar UIs profissionais.

Técnica chamada de UI Sniping: tira prints de sites bonitos (como Dribbble) e fornece como referência para que a IA imite o estilo visual.

🎨 UI Sniping 📸 Referências visuais 🧩 Componentes prontos ✨ Dribbble inspiration
T
Trigger
// DEPLOY
Coloca o app no ar e configura automações que rodam sozinhas na nuvem. Do GitHub ao Vercel, tudo gerenciado pela IA via MCP.

A fase final conecta GitHub → Vercel automaticamente. Variáveis de ambiente e chaves de API são configuradas de forma segura sem exposição manual.

🐙 GitHub auto-commit ▲ Deploy na Vercel 🔐 ENV variables seguras ⚡ Automações na nuvem
// 02 — Ecossistema

As Ferramentas
do Vibecode.

🌐
AntiGravity
// PLATAFORMA PRINCIPAL · GOOGLE
Plataforma principal de desenvolvimento baseada em agentes de IA com modelos Gemini. O ambiente onde tudo é construído conversando com a ferramenta.
🔗
MCP
// MODEL CONTEXT PROTOCOL
Conecta a IA diretamente ao Notion e ao Supabase sem configurar APIs manualmente. O protocolo que elimina friction na integração de serviços.
🧠
Poppy AI
// ANÁLISE DE CONTEÚDO
O "cérebro" para analisar vídeos do YouTube e gerar roteiros e ganchos virais. Transforma conteúdo em insights acionáveis automaticamente.
⚡
Modal.com
// SCRAPING AGENDADO
Roda scripts de raspagem de dados do Reddit e YouTube de forma automática e agendada na nuvem. Coleta de dados 24/7 sem intervenção.
🗄️
Supabase
// MEMÓRIA PERMANENTE
Banco de dados que dá memória permanente à aplicação. Posts do Reddit encontrados pela IA são salvos e persistem entre sessões.
// 03 — As 3 Camadas

Arquitetura
em Camadas.

CAMADA 01
📋
SOPs
// REGRAS TÉCNICAS
Procedimentos operacionais padrão escritos em Markdown. Define as regras técnicas que a IA deve seguir em cada situação — o manual de instruções do sistema.
CAMADA 02
🧠
Navegação
// RACIOCÍNIO DA IA
A camada de tomada de decisão. O raciocínio da IA que decide qual ferramenta usar em cada momento. O "GPS" que conecta intenção à ação correta.
CAMADA 03
⚙️
Ferramentas
// SCRIPTS PYTHON
Scripts Python determinísticos para tarefas específicas. Execução previsível e controlada — sem improviso, sem alucinação, resultados consistentes.

Como Funciona

01
Configuração do Ambiente
Cria uma pasta no AntiGravity e cola o prompt de sistema que define a IA como um "System Pilot" — o ponto de partida de todo projeto.
// Define IA como "System Pilot"
02
Referência de Design
UI Sniping: prints de sites bonitos do Dribbble são fornecidos à IA para que ela imite o estilo visual. Design profissional sem designer.

03
Integração de Dados
Conexão do Supabase via MCP para que a aplicação tenha memória permanente — posts do Reddit encontrados pela IA ficam salvos para sempre.
// Persistência de dados real
04
Ajustes Iterativos
Recursos adicionados em linguagem natural: "adicione upload de foto de perfil" ou "crie uma seção de configurações". Zero código manual.
