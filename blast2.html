// 01 — O Framework

As 5 Etapas do BLAST

B
Blueprint — Plano
// ETAPA 01

Define toda a lógica em linguagem natural antes de escrever código. Responde às 5 perguntas essenciais que orientam o projeto inteiro.

O Blueprint funciona como a fundação do projeto. Sem ele, a IA toma decisões que parecem corretas localmente, mas quebram o sistema globalmente. Pense nele como o contrato entre você e a ferramenta — uma vez definido, toda decisão posterior se ancora nele.

🎯 North Star — objetivo principal 🔗 Serviços externos 🗄️ Fonte da verdade 📦 Formato de entrega 📋 Regras de comportamento
L
Links — Conectividade
// ETAPA 02

Testa todas as conexões de API e credenciais antes de começar a codificar. O "handshake" que garante que serviços externos vão se comunicar.

Validar conexões antes de construir evita horas de debug. Se uma API falha apenas no final do desenvolvimento, todo o esforço investido nas etapas anteriores fica comprometido. Links é o seguro do projeto.

✅ Teste de APIs antes de codificar 🔑 Validação de credenciais 🤝 Handshake verification ⏱️ Referência @ min 09:30
A
Architecture — Arquitetura
// ETAPA 03

Divide o app em três camadas distintas: SOPs, Navegação e Ferramentas. Cada camada tem responsabilidade única — a IA nunca precisa adivinhar.

A separação em camadas é o que torna o sistema previsível e auditável. Cada camada tem uma linguagem própria e escopo bem delimitado, o que reduz alucinações e torna o comportamento da IA determinístico.

📝 Camada 1 — SOPs em Markdown 🧠 Camada 2 — Raciocínio da IA 🐍 Camada 3 — Scripts Python determinísticos ⏱️ Referência @ min 21:31
S
Stylize — Estilização
// ETAPA 04

Refina o design usando referências visuais e componentes prontos. A técnica de UI Sniping transforma prints do Dribbble em interfaces profissionais.

UI Sniping: capture prints de sites que você admira e entregue à IA como referência visual. Ela absorve o estilo — espaçamento, paleta, tipografia — e reproduz fielmente sem que você precise de um designer contratado.

🎨 UI Sniping como método 📸 Referências visuais do Dribbble 🧩 Componentes prontos reutilizáveis ✨ Design profissional sem designer
T
Trigger — Deploy
// ETAPA 05

Coloca o app no ar e configura automações que rodam sozinhas na nuvem. Do GitHub ao Vercel, gerenciado pela IA via MCP sem intervenção manual.

A fase final conecta GitHub → Vercel automaticamente. Variáveis de ambiente e chaves de API são configuradas de forma segura por dentro do AntiGravity via MCP da Vercel — sem abrir um terminal ou expor credenciais.

🐙 GitHub auto-commit ▲ Deploy na Vercel via MCP 🔐 ENV variables seguras ⚡ Automações agendadas na nuvem
// 02 — Ecossistema

As Ferramentas
do Vibecode

🌐
AntiGravity
Plataforma principal · Google

Ambiente de desenvolvimento baseado em agentes de IA com modelos Gemini. O hub central onde Blueprint, código, design e deploy acontecem numa única conversa.

🔗
MCP
Model Context Protocol

Conecta a IA diretamente ao Notion e ao Supabase sem configurar APIs manualmente. Elimina toda a friction de integração — a IA fala diretamente com os serviços.

🧠
Poppy AI
Análise de conteúdo

O cérebro para analisar vídeos do YouTube e gerar roteiros e ganchos virais. Transforma conteúdo existente em insights e scripts prontos para publicação.

⚡
Modal.com
Scraping agendado

Roda scripts de raspagem do Reddit e YouTube de forma automática e agendada. Coleta de dados 24/7 sem qualquer intervenção — o app trabalha enquanto você dorme.

🗄️
Supabase
Memória permanente

Banco de dados que dá memória real e persistente à aplicação. Posts encontrados pela IA, preferências e histórico ficam salvos e disponíveis entre sessões.
